### OLED 残像対策
- pure `#000000` / `background: black` / `background: #000` の grep → **0 件**
- bg-base は `#07050D`（near-black + violet tint）で smear 回避

### a11y（ui-ux-pro-max ガイドライン）
- [x] skip-link `.noxa-skip-link` 実装、`<main id="main">` セマンティクス
- [x] focus-visible グローバル強化（a / button / role=button / tabindex）
- [x] BottomTabBar に aria-current="page" / aria-label
- [x] SalesTrendChart に role="img" + データテーブル fallback `<details>`
- [x] Loading に role="status" + aria-busy
- [x] Error に role="alert" + aria-live
- [x] `prefers-reduced-motion` ガード（3 リポすべて）
- [x] Input min-h 44px + font-size 16px（iOS auto-zoom 回避）

### モバイル動線
- BottomTabBar: 5 タブ、md 以上で hidden、safe-area-inset 対応
- touch target: btn / input すべて 44×44px 以上保証
- LP は `clamp()` で responsive font、`auto-fit / minmax()` で grid 自動展開

### Playwright スクショ
- `_design-source/variants/shots/qa-{375,768,1024,1440}.png` で 4 ブレークポイント記録
- 1280px 固定の artboard preview のため、スマホでは横スクロール（preview HTML の都合）

## 7. yorulog Android（Capacitor + WebView）への波及

```
yorulog/src/app/  (Next.js)
       ↓ deploy
yorulog.vercel.app
       ↓ WebView wrap
yorulog Android (Capacitor, Google Play)
```

- Web の v1.1 改善（特に `RealTimeMetrics` / `SalesTrendChart`）が **そのまま Android アプリに反映される**

## 10. 注意点まとめ

1. **iOS リポ（yorulog-ios）は一切触っていない**
2. yorulog/nomishugy の既存 shadcn / iOS 風 utility は維持（破壊的変更なし）
3. NOXA token は `.noxa-zone` クラスで選択的適用、グローバル汚染なし
4. pure black 不使用ルール（OLED smear 回避）は grep 監査で 0 件確認済み
5. WCAG コントラスト全 token ペア検証済み、本文 < 13px に `text-faint` 禁止のルールあり
6. `prefers-reduced-motion` は 3 リポすべてで有効、glow / texture / animation を全てキャンセル
7. SalesTrendChart は依存ゼロの純 SVG 実装（Recharts のような追加 npm パッケージ不要）
