## 2. WCAG コントラスト結果

```
pair                                    contrast  AA-N  AA-L  AAA-N
text-primary on bg-base                  18.18:1   OK    OK    OK
text-muted on bg-base                     8.07:1   OK    OK    OK
text-faint on bg-base                     3.73:1   NG    OK    NG    ⚠️
text-primary on surface-card             16.23:1   OK    OK    OK
text-muted on surface-card                7.21:1   OK    OK    OK
text-faint on surface-card                3.33:1   NG    OK    NG    ⚠️
white on accent-primary                   4.23:1   NG    OK    NG    ⚠️ → bold 化で AA-L 通過
white on destructive                      5.23:1   OK    OK    NG
ink on bg-base                            8.86:1   OK    OK    OK
neon on bg-base                           7.67:1   OK    OK    OK

info (cyan-mist) on bg-base              13.98:1   OK    OK    OK
warning on bg-base                       14.03:1   OK    OK    OK
```

### 2.1 対応済み: white on accent-primary
- `.noxa-btn` の `font-weight` を 500 → **600** に変更
- WCAG では 14px bold（または 18px normal）以上が「large text」扱いで 3:1 でクリア
- 結果: 4.23:1 のままで AA-L をクリア（実視認性は十分）

### 2.2 残課題: text-faint の使用箇所制限
- `#6E6585` を **本文（13px 未満）に使わない** ことをルール化
- 用途は: footer の © 表記、装飾的キャプション、disabled 風プレースホルダー等
- 必要な場面で本文として使う場合は `text-muted (#A89FBE)` に格上げ

## 3. UX ガイドライン適用ログ

ui-ux-pro-max の検証で重要だった対応:

| ガイドライン | 適用内容 |
|---|---|
| Continuous Animation は loading 専用 | **pulse-glow CTA を削除**、hover-only glow に変更 |
| prefers-reduced-motion 対応 | **3 リポすべての globals.css にガード追加** |
| Touch target 44×44px | `.noxa-btn` に `min-height: 44px`、`.noxa-input` に `min-height: 44px` |
| iOS auto-zoom 回避 | `.noxa-input` の `font-size: 16px`（14px から引き上げ） |
| Status 色 + テキスト二重符号化 | `.noxa-status-*` バッジ utility（ドット + テキスト、glow 付き） |
| Motion natural easing | `--noxa-ease-natural: cubic-bezier(0.16, 1, 0.3, 1)` を追加 |
| OLED smear 回避 | bg-base を `#000000` でなく `#07050D` 維持 |

## 4. 3 リポの統合戦略

```
┌──────────────────┐
│ NOXA design      │  v1.1 master
│ system (token +  │  noxa-platform/_design-source/
│ guidelines)      │
└────────┬─────────┘
         │ mirror
   ┌─────┴──────┬───────────┐
   ▼            ▼           ▼
┌────────┐ ┌─────────┐ ┌──────────┐
│ noxa-  │ │ yorulog │ │nomishugy │
│platform│ │ (sub-   │ │ (sub-    │
│        │ │  brand) │ │  brand)  │
└────────┘ └─────────┘ └──────────┘

  ▲ NOXA token 直適用      ▲ NOXA mirror layer のみ
                            （既存 shadcn / iOS 風スタイル維持）
                            `.noxa-zone` クラスで選択適用可
```

### yorulog / nomishugy で `.noxa-zone` を使う想定箇所
- LP の hero / CTA セクション
- ログイン・サインアップ画面（NOXA アカウント統合への入口）
- 料金プラン画面
- NOXA Community への導線バナー

### 将来の最終統合
NOXA リリース時に yorulog / nomishugy の shadcn token (`--primary`, `--background`, `--card` 等) を NOXA 値で **置換すれば全画面 NOXA 化**。今は mirror layer として並走させているため、置換のタイミングは別判断可能。
